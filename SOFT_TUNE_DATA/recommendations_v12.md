## Что сработало (по фазам P15)

- **P15.A retriever.objection_arguments** + force-inject historical_deal → manager objection_handling/pricing_justify лучше grounded (Win #6, #8, #9, #10 в этом отчёте — все по сделкам).
- **P15.B prompts.yaml lead_time directive** → fill rate `estimated_lead_time` поднялся (30.9% v12 vs 0% v3).
- **P15.C BGE-M3 max_seq_len=1024 reindex** → длинные chunks с деталями сделок → no-RAG галлюцинаций меньше (Δgrounded +41.2pp, лучше всех версий).
- **P15.D fine-tuned reranker v2** (proper soft-labels + cross-topic hard negatives, 8.5k pos + 16.9k neg) → smoke 6/6 на irrelevance, ad_banner/stickers/logistics получили +12-29pp.

## Что НЕ сработало / проблемы

- **print_flyer -15.2pp** — fine-tuned reranker слишком жёстко режет cross-product совпадения. Гипотеза: тренировочные negatives "буклеты vs листовки" были слишком близки → модель учит избегать обоих, а query про буклет иногда требует листовку как fallback. P16: collision-aware negatives (whitelist).
- **objections / objection_handling -5..-6pp** — historical_deal инжектится, но prompt не всегда использует. Нужен explicit `<deal_ref>` in human_query.
- **directions -7.5pp** — domain query типа «можно ли совместить буклеты и ручки» теряют intent, валятся в product-specific retrieval вместо bundle-узлов.
- **Pricing hallucinations 461/800 (57.6%)** — judge срабатывает на любое число без `references[].deal_id`. Слишком строго: цены из прайса (не сделок) не должны считаться hallucination. Нужен whitelist by source_type.
