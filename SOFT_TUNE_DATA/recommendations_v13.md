## Что НЕ сработало

- **print_flyer −9.9pp** — главное разочарование. v3 reranker не справился с буклет/листовка как ожидалось. Гипотеза:
  - soft_neg label=0.3 размыло discrimination
  - test set v13 включал query типа "виды фальцовки", "тип скрепления", "поиск макетов" — это не reranker miss, это **intent miss** (RAG отвечает про стоимость листовок вместо технических вопросов про буклеты)
  - В Fail #1-5 из топ-10 RAG-fails — все print_flyer, причина общая: RAG retrieves "стоимость листовок А6" вместо "техника буклетов"
- **merch −12.6pp** — менее предсказуемо. Возможно widening test set обнажил weak coverage.
- **A.5 deal_items autofill** — только 30.9% priced answers получили deal_items (target был 95%). Причина: refs метаданные (knowledge/faq docs) часто без line_total → fallback не активируется.

## Open issues + P17 candidates

1. **print_flyer fix** (главное!) — intent classifier должен отличать "сколько стоит буклет" (product_query) от "какая фальцовка для буклета" (technical_clarify, новый intent). Plus collision-aware reranker v4 с balanced labels (no soft_neg, only hard binary).

2. **A.5 deal_items strengthening** — расширить metadata в knowledge_docs (добавить price hints) ИЛИ ослабить требования fallback (генерировать из summary text если refs пусты).

3. **merch corpus refresh** — проверить что после max_seq_len=1024 reindex merch chunks правильно retrieval-matchятся. Возможно нужен dedicated merch direction prototype expansion.

4. **Discovery questions** — `m_discovery_049` ("Напишите письмо-запрос") валится — manager задачи на текстовую помощь не должны идти в pricing path. Intent classifier weak.

5. **Pricing_grounded calibration v3** — судья v2 всё ещё помечает 38.4% pricing_grounded RAG (vs 1.1% no_RAG). Это здоровая дельта +37.3pp, но **984 "hallucinations" в репорте** — много false positives на цены из прайса. Возможно требуется ещё один шаг recalibration или explicit ref-marking в outputs.
