### Ключевой инсайт: train/test contamination в v5

v5 показал 66.8%, но использовал те 400 v3 вопросов, на которых **тренировались prototypes** (`build_intent_prototypes.py` читал questions_v3.jsonl как input). Это **in-distribution evaluation**.

v6 использовал **800 NEW questions** (out-of-distribution) → 62.0%. На 3.4pp хуже v4 (OOD vs OOD сравнение).

**Реалистичный baseline текущей системы: ~62%** (не 66.8%).

### Гипотеза причины регрессов

P14.5 prototypes собраны из labelled v1/v2/v3 questions с `expected_intent`. P14.6 rebalance дополнительно **сузил**: describe 60→30, underspec 59→20 — overfit на конкретные query patterns.

На v3 (in-distribution) — система знает паттерны → высокий win-rate. На **новых** v6 — паттерны слегка отличаются, intent classifier ошибается чаще на edge cases (discovery, print_flyer особенно).

### Recommendations P14.7

1. **Расширить describe/underspec back до 40-50 каждый** — балансировка между специфичностью и breadth. P14.7.A: tweak `rebalance_prototypes.py` чтобы не cap'ить агрессивно.

2. **Transfer labus_docs_v7 (max_seq_len=1024) на прод** — locally built (15:54), 62964 points. Передать embeddings.npy через scp, на проде `build_index.py --from-npy --collection labus_docs_v7`. Может частично восстановить discovery/print_flyer.

3. **Hybrid eval going forward** — всегда mix in-distribution (v3) + OOD (new). Метрика честнее.

4. **HyDE / query rewriting** — для discovery/print_flyer queries. Generate ideal_response → embed → retrieve. +1-2s latency, +3-8% recall.

5. **Reranker fine-tune** на накопленных 6400+ парах (v1+v2+v3+v4+v5+v6 scores) — самый дорогой но эффективный шаг (P15).
